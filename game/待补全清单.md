## 四、优先级排序

```
P0 —— 阻断可玩性
   1. 化神~渡劫敌人补全（否则高境界零战斗）
   2. 玄阶/仙阶功法补全（否则学习停滞）

P1 —— 缺少体系
   3. 炼丹/炼器基础框架（材料→产物，craftBonus 生效）
   4. 宗门贡献系统（contribRate → 灵石，做任务攒贡献）
   5. 副本/秘境（多波次战斗 + 宝箱结算）

P2 —— 增加厚度
   6. 奖励表补全（4张）
   7. 拍卖行/交易系统
   8. 食物/典籍物品扩展
   9. NPC 好感度系统
   10. 称号/成就系统

P3 —— 锦上添花
   11. worldNames 宗门动态化
   12. 区域专属开局剧情
   13. 灵兽/宠物
   14. 师徒/道侣
   15. PvP/排名
```

## 五、代码层需要新增的 API / 方法（对照 P0-P2）

```
state.py:
  - def sect_contribution(self, amount) → int    # 增减贡献度
  - def craft_calc(self, formula_id) → dict      # 炼制成功率/产出
  - def dungeon_enter(…) / dungeon_step(…)       # 副本状态机
  - class NPCState:                               # NPC 关系持久化
        affinity: int, flags: list, met: bool

engine.py:
  - def craft(self, recipe_id) → scene           # 触发炼制
  - def enter_dungeon(self, dungeon_id) → scene  # 进入副本

server.py (API):
  - POST /api/craft          → {ok, product, state}
  - POST /api/auction        → {ok, bid, state}
  - POST /api/dungeon        → {ok, battle_chain, state}
  - GET  /api/achievements   → {list}
```
